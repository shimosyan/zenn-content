## これはなに？

こんにちは、しもしゃんです。

私には趣味で運営している同人サークルのホームページがあります。

もともとは WordPress で動かしていましたが、コスト（約 3,000 円/月）が地味に重たくなんとかしたいと思ってました。

ですが、今年ついに長年連れ添った WordPress を卒業し、静的サイトジェネレータ（Hugo）へ移行。
結果として**月額コストは数十円（約 1/100）**になり、さらにコンテンツ管理が簡単になりました！

WordPress といえば、情シス目線ではプラグインの脆弱性管理が気になるところ…。今回の移行はその管理からも解放されて嬉しいポイントばかりです。

今回は、技術的なこだわりと少しの泥臭い苦労を交えて、その移行ログを共有したいと思います。

## HP の歴史

まずは、私のサイトがどのような変遷を辿ってきたか、軽く振り返ります。

2013 年（自宅サーバー期）： 物理サーバーで WordPress 運用。自由度は最高でしたが、ハードウェアの故障に怯えながら、ミドルウェアの保守に苦しむ日々。この頃のコストは電気代約 3,000 円/月。

2017 年（VPS 期）： 自宅サーバーを襲った突然の事故(※)に伴い一般的な VPS へ移行。ハードウェア管理がなくなり楽にはなりましたが、ミドルウェアの保守の辛さは解消されず。この頃のコストは VPS 代の約 3,400 円/月。

2022 年（Headless CMS 期）： ミドルウェアの依存性の複雑さが限界に。フロントエンド主体の流行りに乗る形で、Headless WordPress が使える某サービスへ乗り換え。導入当時は月額約 1,500 円弱と満足していましたが、円安や値上げの影響でコストが月額 3,000 円近くまで上昇。

2025 年（現在、本件）： ベンダーロックインのリスクを感じ、コストに耐えかねて Hugo + AWS へ移行。

「月 3,000 円なら安いもの」という考え方もありますが、「同じくらいのコスト払うならレンタルサーバーとか VPS とか EC2 の方が自由度あるじゃん。でもミドルウェアの保守は二度とやりたくない」と課題に思ってました。

そんな課題を会社のチームメンバーに話したところ、Hugo というツールのことを教えてもらいました。概要を調べてみると、「ウチの HP をこれで運用できたら最高じゃん」と感じて Hugo への乗り換えを決意しました。

## 新しいアーキテクチャの全体像

移行にあたって採用した構成は以下の通りです。

- SSG: Hugo（ビルド速度と自由度で選定）
- Hosting: AWS S3 + CloudFront
- CDN/DNS: Cloudflare
- CI/CD: GitHub Actions
- IaC: Terraform

### こだわりポイント：メディアファイルの分離

Hugo のリポジトリが画像ファイルで肥大化するのを避けるため、静的 HTML 用の S3 バケットとは別に、メディアファイル専用の S3 バケットを用意しました。

記事の Markdown ファイルは GitHub で管理し、画像は S3 に直接アップロードする運用です。
これでリポジトリも軽量に保てます。

### 補足

通常は CloudFront + Route53 が王道ですが、ドメイン管理を Cloudflare で行っていたこともあり、あえて Cloudflare -> CloudFront -> S3 という構成にしました。
